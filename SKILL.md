# 模拟操盘策略

## 概述

严格按本策略执行每日4个固定时间点巡查，模拟A股交易。

- **初始资金：** 10万元
- **当前持仓：** 空仓（见 portfolio.md）
- **巡查时间：** 09:35 / 11:25 / 13:05 / 14:55

---

## 核心规则

### 选股逻辑
- **资金流选股**：先用东财API扫全市场板块资金流，按主力净流入排序，选前3-5名作为候选主线
- **板块内选龙头**：在候选板块内按个股资金流选前2-3只，剔除ST/成交额<5000万/当日下跌的
- **消息面验证**：候选板块必须有当日政策或产业利好消息才考虑建仓
- **优先龙头 + 基本面拐点 + 机构加仓 + 利空出尽**，不做ST/小盘纯炒作/没业绩冷门股

### 仓位规则
| 市场环境 | 总仓位 | 单票仓位 | 最多持仓 |
|---------|--------|---------|---------|
| 震荡行情 | 40%–60% | ≤30% | 2只 |
| 主线明确+双利好 | 70%–90% | ≤35% | 3只 |
| 大盘走弱/主线崩塌 | ≤20%或空仓 | ≤20% | 1只或空仓 |

- 同一时间最多持仓3只
- 最低单票仓位15%（至少1.5万）

### 买卖规则
- **买入**：回调低吸、震荡企稳、利空落地、均线支撑位分批建仓，**绝不追涨**
- **加仓**：盈利后趋势确认才加，亏损**绝不补仓摊成本**
- **止盈**：目标压力位、情绪过热、板块分化，分批减仓
- **止损**：跌破关键支撑、逻辑破坏，果断小亏离场

### 止损止盈
- **固定止损**：单笔亏损 ≥8% 强制止损
- **移动止损**：盈利后以最新价5%设置动态止损
- **分批止盈**：目标压力位分2批减仓（30%+30%）
- **盈亏比原则**：止盈空间 ≥ 止损空间的2倍，否则不开仓

### 三层决策框架（天时-地利-人和）

**天时（值不值得参与）**
- 大盘位置：沪指在5日线上方 → 积极；跌破5日线 → 谨慎
- 宏观政策：政策强催化 → 重仓；无催化 → 轻仓
- 市场情绪：成交额 ≥2万亿 → 活跃；＜1.5万亿 → 低活跃

**地利（站在哪最有优势）**
- 板块动量：5日涨幅排名前3的主线板块
- 资金合力：主力净流入板块，跟随不预判
- 龙头效应：只做龙头，板块中军稳定则安心持有

**人和（执行纪律）**
- 规则先于感觉，避免主观臆测
- 连续2次止损后，强制休息1天
- 尾盘（14:30后）不新开仓，只做减仓或持有

### 绝对禁止的行为
- ❌ 盘中突然拉高（已涨超5%）不追
- ❌ 成交额低于5000万的股票不买
- ❌ 消息未经证实不下单
- ❌ 连续亏损2次后强行加仓
- ❌ 尾盘最后10分钟新开仓
- ❌ **严禁虚构价格** — 所有数据必须用真实行情，没有数据就不操作
- ❌ **同一时间持仓超过3只**
- ❌ **卖出一只后3个交易日内买回同一只**（冷却期）
- ❌ **做无业绩纯情绪连板妖股**

### 空仓休息规则（强制触发）
以下任意一条触发时，强制总仓位≤10%或空仓，当日禁止新开仓：
- 沪指跌破5日线 **且** 成交额连续2天＜1.5万亿
- 主线板块集体大跌、龙头股跌停潮（≥3只主线龙头跌停）
- 当日已发生单日账户总亏损 ≥3%

### 风控规则
- 单日账户总亏损 ≥3% → 当日停止买入，只持仓或减仓
- 连续2日累计回撤 ≥5% → 次日强制空仓休息一天
- 触发后必须记录原因，明确告知老板

### 持仓过夜规则
- 节前最后一个交易日：总仓位主动降20%
- 板块连续大涨3天+高位放量滞涨：尾盘主动减仓
- 周五尾盘：无明确催化剂，仓位控制在30%以内

### 轮动规则
- 同一板块连续大涨2天后，**不再追高新开仓**
- 轮动到新主线时，优先切换到新主线龙头

### 建仓分批规则
- 单只个股分**2批**建仓，不一次性梭哈
- 第一批15%–20%，趋势确认后次日补剩余仓位
- 两批间隔至少1个交易日
- 建仓期间下跌≤8% → 止损离场

## 每日巡查流程

每个时间点执行以下步骤：

1. **读取持仓记录** `skills/stock-trading/portfolio.md`
2. **读取当日日志** `skills/stock-trading/logs/YYYY-MM-DD.md`
3. **【全市场板块资金流扫描】** 东财API查全市场，按主力净流入排序取前3-5名
4. **【板块内选龙头】** 各候选板块内按资金流选前2-3只
5. **【消息面验证】** 搜索当日政策/产业消息，有利好才考虑建仓
6. **【获取实时价格】** 新浪API批量查候选个股行情
7. **【仓位决策】** 持仓不动 / 买入 / 加仓 / 减仓 / 止损
8. **【记录日志】** 操作时间、决策依据、仓位变动
9. **【更新持仓】**

### 消息面获取方式
```bash
# 方式一：web_search（快，~3秒）
web_search "2026年5月8日 A股 政策 利好 利空"

# 方式二：浏览器豆包（综合分析）
# 打开豆包，输入查询内容，获取综合分析
# 具体操作方式见 browser-automation/SKILL.md 类型一
```

### 巡查决策规则
- 有持仓时：检查是否触发止损/止盈条件
- 无持仓时：按「初始建仓流程」5步选股
- 每次操作前必须校验：是否触发空仓休息规则、单日回撤限制

## 初始建仓流程（5步，必须按顺序执行）

**核心原则：先看全市场资金流，再定板块，最后选个股。不要预设板块。**

### 第一步：选板块 — 全市场板块资金流排名
```bash
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=50&pn=1&np=1&fltt=2&invt=2&fs=m:90+t:2+f:!50&fields=f12,f14,f62,f184"
```
- fid=f62：按主力净流入排序
- f12=板块代码，f14=板块名称，f62=主力净流入额（元），f184=净流入占比（%）
- 取前3-5名作为当日候选主线

### 第二步：选个股 — 板块内按资金流选龙头
```bash
# 以某板块BK1301为例
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=15&pn=1&np=1&fltt=2&invt=2&fs=BK1301&fields=f12,f14,f62,f184,f69,f72"
```
- 各候选板块内选主力净流入最大的2-3只
- 排除：ST、小盘（成交额<5000万）、当日下跌

### 第三步：消息面验证
```bash
# 各候选板块都搜一遍
web_search "2026年5月8日 游戏板块 政策 利好"
web_search "2026年5月8日 锂电池 产业 消息"
```
- 有政策/产业利好 → 可考虑建仓
- 无消息或有利空 → 剔除

### 第四步：获取候选个股真实行情
```bash
curl -s "https://hq.sinajs.cn/list=shxxxxx,szxxxxx,..." -H "Referer: https://finance.sina.com.cn"
```
- 筛选条件（必须同时满足）：
  - 涨跌幅 > 0（顺势而为）
  - 成交额 ≥ 5000万
  - 现价相比开盘价涨幅 ≤ 5%（不追高）

### 第五步：仓位计算 + 规则校验后执行
- 总仓位上限90%（主线明确时），单票≤35%
- 第一批建仓15%–20%，次日补剩余仓位
- 计算实际可买股数（100股整数倍）
- 记录选股理由（附资金流数据支撑）

## 实时行情获取

### 方式一：新浪财经 HQ 接口（首选，推荐）
```bash
curl -s "https://hq.sinajs.cn/list=sh000001,sz399001,sz399006,sh000688" -H "Referer: https://finance.sina.com.cn"
```
- 字段：名称,今开,昨收,现价,最高,最低,买一价,卖一价,成交量,成交额,...

### 方式二：东方财富 API（JSON格式，用于精确数据）
```bash
# 单股行情
curl -s "https://push2.eastmoney.com/api/qt/stock/get?secid=1.688041&fields=f43,f57,f58,f169,f170,f44,f45,f46,f60"

# 板块资金流
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=30&pn=1&np=1&fltt=2&invt=2&fs=m:90+t:2+f:!50&fields=f12,f14,f62,f184"
```

### 方式三：浏览器豆包（综合分析，需要时用）
当API数据不完整、或需要综合分析市场时用。具体操作方式见 `browser-automation/SKILL.md` 类型一：AI对话机器人。

### 数据优先级
| 需求 | 推荐方式 | 耗时 |
|------|---------|------|
| 实时股价/指数 | 新浪API | ~2秒 |
| 板块资金流排名 | 东财API | ~3秒 |
| 综合分析 | 浏览器豆包 | ~30秒 |
| 查不到数据 | **如实告知老板，暂停操作** | — |

### 数据使用规则
- 每次操作前必须获取最新实时价格
- 计算仓位以实际成交价格为准
- **无数据不操作**，查不到价格时暂停当日操作
- 数据不完整时，明确告知老板哪些拿到了、哪些没拿到

## 行情接口容错规则

1. 新浪API / 东财API **拉取失败** → 自动切换其他API或浏览器豆包查询
2. 所有渠道都拿不到有效数据 → **当日暂停所有买卖**，记录"无行情数据，无法操作"
3. 任何数据缺失，必须在日志里写明原因
