# 模拟操盘策略

## 概述

严格按老板制定的「专属模拟操盘总策略」执行，每日4个固定时间点巡查，模拟A股交易。

初始资金：**10万元**
当前持仓：空仓
基准日期：2026-05-07

---

## 核心规则（绝对遵守）

### 1. 选股逻辑
- **主线由实时消息决定**：通过最新市场消息、资金动向、板块情绪来识别当前最强主线，不预设行业偏好
- 次选方向（当前市场偏好参考）：AI算力、半导体芯片、信创国产替代、科创硬科技、新能源、消费电子等
- 优先龙头 + 基本面拐点 + 机构加仓 + 利空出尽
- 不做ST、小盘纯炒作、没业绩冷门股

### 2. 仓位规则
| 市场环境 | 总仓位 | 单票仓位 | 最多持仓 |
|---|---|---|---|
| 震荡行情 | 40%–60% | ≤ 30% | 2只 |
| 主线明确+双利好 | 70%–90% | ≤ 35% | 3只 |
| 大盘走弱/主线崩塌 | ≤20% 或空仓 | ≤ 20% | 1只或空仓 |
- **同一时间最多持仓3只个股**，绝不超过
- **最低单票仓位15%**（至少1.5万，否则没必要买）

### 3. 买卖规则
- **买入**：回调低吸、震荡企稳、利空落地、均线支撑位分批建仓，**绝不追涨**
- **加仓**：盈利后趋势确认才加，亏损**绝不补仓摊成本**
- **止盈**：目标压力位、情绪过热、板块分化，分批减仓落袋
- **止损**：跌破关键支撑、逻辑破坏，果断小亏离场，不扛深套

### 4. 止损止盈规则（强化版）
- **固定止损**：单笔亏损 ≥ 8% 强制止损，不扛深套
- **移动止损**：盈利后，以最新价的 5% 设置动态止损（锁住利润）
- **分批止盈**：目标压力位分 2 批减仓（30% + 30%），不贪最高点
- **盈亏比原则**：止盈空间 ≥ 止损空间的 2 倍，否则不开仓

### 5. 三层决策框架（天时-地利-人和）
**天时（值不值得参与）**
- 大盘位置：沪指在5日线上方 → 积极；跌破5日线 → 谨慎
- 宏观政策：政策强催化 → 重仓；无催化 → 轻仓
- 市场情绪：成交额 ≥ 2万亿 → 活跃市场；＜1.5万亿 → 低活跃

**地利（站在哪最有优势）**
- 板块动量：5日涨幅排名前3的主线板块
- 资金合力：主力净流入板块，跟随不预判
- 龙头效应：只做龙头，板块中军稳定则安心持有

**人和（执行层面的纪律）**
- 规则先于感觉，避免主观臆测
- 连续2次止损后，强制休息1天，不强行交易
- 尾盘（14:30后）不新开仓，只做减仓或持有

### 6. 买前参考指标（参考权重，不做硬性门槛）
1. **均线结构**：多头排列（5/10/20日均线向上）为加分项
2. **成交量**：近期明显放量（量能放大1.5倍以上）为参考信号
3. **K线位置**：近期低位为加分项，高位追涨需谨慎
4. **市值**：优先中等规模以上（成交额≥5000万）

**注**：以上为参考权重，不做硬性排除，综合判断可能盈利时可选择性忽略。

### 7. 绝对禁止的行为
- ❌ 盘中突然拉高的股票不追（已涨超5%再拉不动）
- ❌ 成交额低于5000万的股票不买（流动性风险）
- ❌ 消息未经证实不下单
- ❌ 连续亏损2次后强行加仓
- ❌ 尾盘最后10分钟新开仓
- ❌ **严禁虚构价格、凭想象填数据** — 所有持仓成本、市值、盈亏必须使用真实行情数据，没有数据就不操作，绝不捏造
- ❌ **数据不足时不得胡编** — 查不到就说查不到，明确告知老板哪些数据有、哪些没有，绝不能拼凑推测数据来应付
- ❌ **同一时间持仓超过3只** — 最多3只，避免精力分散顾不过来
- ❌ **卖出一只后3个交易日内买回同一只** — 调仓冷却期，避免来回反复打脸
- ❌ **做无业绩纯情绪连板妖股** — 只做趋势中军和主线龙头，不做纯炒作妖股

### 8. 空仓休息规则（强制触发）
以下任意一条触发时，**强制总仓位≤10%或空仓**，当日禁止新开仓：
- 沪指跌破5日线 **且** 成交额连续2天＜1.5万亿
- 主线板块集体大跌、龙头股跌停潮（≥3只主线龙头跌停）
- 当日已发生单日账户总亏损 ≥3%

### 9. 单日最大回撤风控
- **单日账户总亏损 ≥3%** → 当日停止一切买入，只持仓或减仓
- **连续2日账户累计回撤 ≥5%** → 次日强制空仓休息一天
- 触发后必须记录原因，明确告知老板

### 10. 持仓过夜规则
- **节前最后一个交易日**：总仓位主动降20%，不高仓位赌假期消息
- **板块连续大涨3天+高位放量滞涨**：尾盘主动减仓，不重仓过夜
- **周五尾盘**：若无明确催化剂，仓位控制在30%以内

### 11. 轮动规则
- 同一板块连续大涨2天后，**不再追高新开仓**，只持有不新买
- 轮动到新主线时，优先切换到新主线龙头，卖出老主线持仓

### 12. 建仓分批规则（强制）
- 单只个股分 **2批** 建仓，不一次性梭哈
- 第一批：15%–20%仓位，趋势确认后补剩余仓位
- 两批之间间隔至少 **1个交易日**（不当天追着补）
- 建仓期间个股下跌 ≤8% → 止损离场，不等第二批

### 13. 行情接口容错规则
1. 新浪API / 东财API **拉取失败** → 自动切换浏览器豆包综合查询
2. 豆包也拿不到有效行情（超时/内容为空）→ **当日暂停所有买卖**，只记录"无行情数据，无法操作"
3. 任何一步数据缺失，必须在日志里写明原因，不得用推测数据代替

### 14. 操作风格
- 波段中线，拿得住主升浪
- 不天天频繁买卖
- 每天根据盘前消息、板块情绪、龙头强弱微调仓位
- 新开仓必须分2批，不梭哈
- 同一板块不连续追3天，轮动切换有节奏

## 每日巡查流程

每个时间点（09:35 / 11:25 / 13:05 / 14:55）执行：

1. **读取持仓记录** `skills/stock-trading/portfolio.md`
2. **读取当日日志** `skills/stock-trading/logs/YYYY-MM-DD.md`
3. **天时判断（消息面）**：搜索今日重大新闻/政策，找出影响A股的宏观因素
4. **地利判断**：用东财API查询板块资金流排名，结合消息确认主线方向
5. **人和决策**：获取候选个股实时价格，用数据筛选，按规则执行
6. **仓位决策**：持仓不动 / 买入 / 加仓 / 减仓 / 止损
7. **记录日志**：操作时间、决策依据、仓位变动
8. **更新持仓**

**天时判断（消息面）标准流程：**
```bash
# 方式一：web_search（快，~3秒）
web_search "2026年5月7日 A股 政策 利好 利空 重大消息"
web_search "2026年5月 科创板 半导体 AI 政策动态"

# 方式二：浏览器豆包（综合分析，~30秒）
smart-open.sh https://www.doubao.com
# 输入："今日A股有哪些影响市场的重要新闻、政策或宏观事件？请列出标题和核心内容"
# 用evaluate提取回复
```

**消息面的判断规则：**
- 有重大政策利好（降准、补贴、扶持政策）→ 积极，可加重仓
- 有宏观利空（美股暴跌、地缘冲突、加息预期）→ 谨慎，减仓
- 无重大消息但资金面强 → 技术性行情，轻仓参与
- **新闻用于判断方向，数据用于验证强度，两者结合才能下单**

**巡查决策规则：**
- 有持仓时：检查是否触发止损/止盈条件
- 无持仓时：按「初始建仓流程」5步选股
- 每次操作前必须校验：是否触发空仓休息规则、单日回撤限制

### 操作日志标准字段（每条必填）
每次操作记录必须包含以下所有字段：
```
## [时间] 操作类型
- 操作时间：HH:MM

```

## 初始建仓流程

**⚠️ 核心原则：数据优先，新闻辅助，消息面+资金面+价格面三层验证后才能下单。**

### 标准化选股流程（5步，必须按顺序执行）

**第零步：消息面扫描（不能跳过）**
> 新闻确定方向，数据验证方向，两者缺一不可
```bash
# 快速扫描今日重大消息
web_search "2026年5月7日 A股 重大政策 利好 利空 产业消息"
web_search "2026年5月 半导体 AI 新能源 政策扶持 行业动态"
```
- 记录：今日有哪些可能影响A股的宏观/行业/个股消息
- 判断：哪些板块受益，哪些板块受损
- 注意：如果搜索结果都是旧闻（几天前的消息），说明当日无重大新催化，更依赖数据面判断

**第一步：选板块 — 用资金流数据排名**
```bash
# 东财板块资金流排名（按主力净流入排序）
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=30&pn=1&np=1&fltt=2&invt=2&fs=m:90+t:2+f:!50&fields=f12,f14,f62,f184"
```
- fid=f62：按主力净流入排序
- f12=代码，f14=名称，f62=主力净流入额，f184=主力净流入占比
- 取前3名板块作为候选主线

**第二步：选个股 — 在选定板块内按资金流排名**
```bash
# 查询板块内个股资金流（示例：半导体板块BK0471）
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=20&pn=1&np=1&fltt=2&invt=2&fs=m:0+t:6,f:!50&fields=f12,f14,f62,f184,f69,f72"
```
- 在板块内选主力净流入最大的2-3只
- 排除ST、小盘（成交额<5000万）

**第三步：获取候选个股真实行情**
```bash
# 批量查询个股实时价格
curl -s "https://hq.sinajs.cn/list=sh688041,sh688256,sh603986" -H "Referer: https://finance.sina.com.cn"
```
- 筛选条件：
  - 涨跌幅 > 0（顺势而为，只买涨的）
  - 成交额 ≥ 5000万（流动性）
  - 不追高：现价相比今日开盘价涨幅 ≤ 5%（回调低吸原则）

**第四步：计算仓位 + 规则校验后执行**
- 按总仓位上限90%（主线明确时）、单票≤35%分配
- 第一批建仓15%–20%，趋势确认后次日再补剩余仓位
- 计算实际可买股数
- 记录选股理由（附资金流数据）
- **全部通过后才执行**

### 新闻的角色
- 新闻用于**判断方向**（哪个板块有机会）
- **不能**仅凭新闻描述就决定买哪只股票
- 必须用资金流数据+实时价格做二次验证

## 实时行情获取

**⚠️ 每次巡查前必须获取真实行情数据，严禁凭想象填数据。查不到就是查不到，必须如实告知老板，不得胡编数据。**

### 方式一：新浪财经 HQ 接口（最快，推荐首选）
```bash
curl -s "https://hq.sinajs.cn/list=sh000001,sz399001,sz399006,sh000688,sh688041,sh688256,sz300750,sh603986" -H "Referer: https://finance.sina.com.cn"
```
- 多股票用逗号分隔，sh=沪市，sz=深市
- **字段顺序**（逗号分隔）：名称,今开,昨收,现价,最高,最低,买一价,卖一价,成交量,成交额,...
- **返回示例**：
  ```
  var hq_str_sh000001="上证指数,4168.45,4160.17,4168.45,4173.76,4163.14,...";
  var hq_str_sh688041="海光信息,342.69,338.40,338.38,343.00,333.88,...";
  ```
- **指数代码**：sh000001=上证指数，sz399001=深证成指，sz399006=创业板指，sh000688=科创50

### 方式二：东方财富 API（JSON格式，推荐用于精确数据）
```bash
# 单股精确行情
curl -s "https://push2.eastmoney.com/api/qt/stock/get?secid=1.688041&fields=f43,f57,f58,f169,f170,f44,f45,f46,f60"
# 板块资金流排名
curl -s "https://push2.eastmoney.com/api/qt/clist/get?fid=f62&po=1&pz=30&pn=1&np=1&fltt=2&invt=2&fs=m:90+t:2+f:!50&fields=f12,f14,f62,f184"
```
- **secid格式**：沪市=1.前缀（如688041→1.688041），深市=0.前缀（如000001→0.000001）
- **字段含义**：f43=最新价，f44=最高价，f45=最低价，f46=今开，f57=代码，f58=名称，f60=昨收，f169=涨跌额，f170=涨跌幅

### 方式三：浏览器豆包查询（用于获取综合分析）
当API数据不完整、或需要综合分析时使用：
1. `smart-open.sh https://www.doubao.com` 打开豆包
2. `snapshot --efficient` 获取输入框 ref
3. `type <ref> "请给出今日A股最新行情：上证指数、深证成指、创业板指、科创50的涨跌幅；海光信息(688041)和寒武纪(688256)的当前股价、今日涨跌情况；今日最强主线板块和龙头股。" --submit`
4. `sleep 12` 等AI生成
5. `evaluate --fn "document.body?.innerText?.slice(0,6000)"` 提取回复内容

> 💡 豆包综合能力强，但耗时约30秒。快速查价格用新浪API（2秒），需要分析时用豆包。

### 数据优先级规则
| 需求 | 推荐方式 | 耗时 |
|------|---------|------|
| 实时股价/指数 | 新浪API curl | ~2秒 |
| 板块资金流排名 | 东财API | ~3秒 |
| 综合分析（主线+龙头+消息） | 浏览器豆包 | ~30秒 |
| 查不到数据时 | **如实告知老板，暂停当日操作** | — |

### 行情数据使用规则
1. **每次操作前**：必须获取最新实时价格
2. **计算仓位**：以实际成交价格为准，不能预估
3. **止损止盈**：以获取到的真实价格计算
4. **无数据不操作**：查不到价格时，暂停当日操作
5. **数据不完整时**：明确告知老板哪些数据拿到了、哪些没拿到，不得拼凑或推测
